# AGENTS.md — инструкция для ИИ-ассистента

Это **«Торговод · Отгрузки FBO»** — локальное веб-приложение (Python + FastAPI) для планирования
поставок Ozon FBO. Твоя задача — развернуть его на компьютере пользователя и провести его по
вводу ключей Ozon. Пользователь **нетехнический** — объясняй простыми словами, по шагам, по-русски.

## Стек

- Python **3.11+**, FastAPI, Uvicorn, SQLite (создаётся сама). Зависимости — в `requirements.txt`.
- Запуск: `uvicorn dashboard.app:app --port 4000` из корня проекта.
- Данные хранятся локально в `data/*.db`. Ключи Ozon — в `.env` (в репозитории его нет, есть `.env.example`).

## Что сделать (по шагам, останавливайся и показывай результат после каждого)

1. Проверь, что установлен **Python 3.11+** (`python --version`). Если нет — подскажи, как поставить под ОС пользователя.
2. Создай виртуальное окружение и активируй его:
   - Windows: `python -m venv .venv && .venv\Scripts\activate`
   - Mac/Linux: `python3 -m venv .venv && source .venv/bin/activate`
3. Установи зависимости: `pip install -r requirements.txt`
4. Создай `.env` из `.env.example`. Затем **по очереди** спроси у пользователя `OZON_CLIENT_ID` и `OZON_API_KEY`,
   коротко объясняя, что это и где взять (кабинет продавца Ozon → Настройки → Seller API → создать ключ,
   роль *Admin read only*). Впиши значения в `.env` **сам**. ⚠️ **Никогда не выводи значения ключей в чат/терминал.**
5. Запусти приложение: `uvicorn dashboard.app:app --port 4000`
6. Дай пользователю ссылку **http://localhost:4000** и скажи открыть её в браузере. На первом экране он сможет
   проверить/ввести ключи, затем нажать **«Обновить данные»** — приложение подтянет данные из Ozon.
7. **Сделай запуск удобным на будущее.** На **Windows** создай ярлык на рабочем столе:
   `powershell -ExecutionPolicy Bypass -NoProfile -File tools\make_shortcut.ps1`
   Объясни: теперь достаточно дважды кликнуть по ярлыку **«Торговод FBO»** — он сам погасит старый
   сервер, запустит приложение и откроет браузер. PowerShell больше открывать не нужно.
   (Ярлык также создаётся автоматически, если пользователь запускает `start.bat`.)

## Можно показать на демо-данных

Если у пользователя пока нет ключей — предложи `python seed_demo.py` (наполнит примерными данными), затем запусти приложение.

## Запреты

- ❌ Не коммить `.env` и файлы `data/*.db` (в них ключи и данные пользователя). Они уже в `.gitignore`.
- ❌ Не выводи реальные значения ключей Ozon в чат.
- ❌ Не придумывай значения ключей сам — всегда спрашивай у пользователя.
- ❌ Не меняй бизнес-логику в `modules/` без явной просьбы — для запуска это не нужно.
